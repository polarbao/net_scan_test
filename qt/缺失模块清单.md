# Qt 项目缺失功能清单

根据对 MFC 源代码 `NetScanDemo` 和当前 Qt 项目的对比分析，以下是 Qt 版本中缺失的核心功能模块：

### 1. 缺失的对话框

以下对话框在 MFC 版本中存在，但在 Qt 版本中完全缺失（没有对应的 UI 文件和类）。主窗口中调用它们的按钮点击事件是空的。

- **温度电压设置 (`CVTSetDlg`)**: 用于管理和设置打印头温度和电压的界面。
- **波形加载 (`CReloadWaveDlg`)**: 用于选择和加载打印头波形文件的界面。
- **运动控制 (`CMoveCtlDlg`)**: 提供对打印平台 X/Y/Z 轴的手动控制功能。
- **气墨系统控制 (`CAirInkDialog`)**: 用于控制负压、墨水泵、清洗等操作的界面。
- **校准与补偿 (`CAdjParamDlg`)**: 用于设置打印的物理校准参数，如步进、喷头间距等。
- **缺嘴补偿 (`CLackJetDlg`)**: 在校准对话框中启动，用于定义和补偿缺失的喷嘴。
- **ADIB 板控制 (`CAdibCtrlDlg`)**: 一个专用的控制/调试对话框。

### 2. 未完整实现的功能

- **图像数据加载 (`getSrcData`)**: `MainWindow::getSrcData` 函数虽然存在，但内部没有实现对 `.prt` 和 `.cli` 文件格式的解析逻辑。目前仅有文件选择的框架，无法真正加载打印数据。
- **图层设置 (`CImgLayerSetDlg`)**: MFC 版本中用于设置每个打印图层的详细参数（如分辨率、颜色、方向等）的对话框在 Qt 版本中没有对应的实现。当前的 `PrintJobDialog` 只包含了最基本的作业设置。

### 总结

当前 Qt 项目完成了基础框架和部分核心功能的移植，但所有与精细硬件控制、高级参数设置和校准相关的功能模块均未实现。下一步工作的重点是逐个创建这些缺失的对话框，并实现其后端逻辑。
